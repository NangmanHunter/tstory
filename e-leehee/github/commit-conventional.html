• Type/Prefix
  • 설명
  • feat
  • 새로운 기능을 추가할 때 사용합니다.
  • fix
  • 버그를 수정할 때 사용합니다.
  • docs
  • 문서에 변경 사항이 있을 때 사용합니다 (코드 변경은 없음).
  • style
  • 코드의 포매팅 변경, 세미콜론 누락, 공백 추가 등 (기능 변경 없음).
  • refactor
  • 리팩토링할 때 사용합니다 (기능 추가나 버그 수정 없음).
  • test
  • 테스트 코드 추가나 수정 시 사용합니다.
  • chore
  • 빌드 프로세스 또는 패키지 매니저 설정 등 기타 변경 시 사용합니다.
  • perf
  • 성능 향상을 위한 코드 변경 시 사용합니다.
  • ci
  • CI 설정 파일 및 스크립트 수정 시 사용합니다.
  • build
  • 빌드 시스템이나 외부 종속성 수정 (예: webpack, npm)
  • revert
  • 이전 커밋을 되돌릴 때 사용합니다.
